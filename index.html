Greenhouse System

Intent

This React web application was developed for the semester project. The project focuses on combining three teams: frontend, backend, and IoT. The purpose of the project is to create web application for displaying temperature, humidity and Co2 levels in a greenhouse. It has also a feature for toggling and scheduling the water system and creating custom presets for vegetables with specific thresholds. This application is connected using REST to Backend part of the project, which is eventually communicating with real IoT sensors and servo motor.

Process

Entire Greenhouse application consists of 3 projects, which are Frontend, Cloud and IoT. Each part was developed by specific sub-team and each sub-team consisted of 4 people. So, the entire applicatio was developed by 12 people. Each sub-team was actively communicating with each other and we used advanced SCRUM framework for large team, which is called LeSS framework.

Rack-Verify

Intent

Rack-Verify is an application that was developed as a Bachelor Project for company Schneider Electric in Denmark. The purpose of the project is to help operators in data centers who are already using software developed by Schneider Electric for modelling data centers. Such as software allows users to create a digital twin of devices in data centers such as servers, racks, rows, etc.

However, currently there isn't any tool that would allow operators to compare real devices in data centers with their digital representation. The only possible way is to compare them manually, which makes the process slow and there is still a huge chance of making a mistake.

Solution

We decided to create Rack-Verify mobile app that allows operators to capture photo of a Rack in data center and then is used machine learning model that is running on the backend for analyzing devices in the rack. On the frontend, the recognized devices are highlighted in the photo and visualized in a 2D Rack model.

The Rack-Verify was developed by 3 people. I was working on the frontend and 2 other guys were working on the backend and machine learning part. Frontend was developed by using React Native with Expo.

Photoguard📸🛡️🌍

Video Demonstration

About the project

The idea for the project came from the desire to contribute to solving the 17 Global Goals presented by the UN in 2015. One of them, the eleventh goal called "Sustainable cities and communities", forces the inhabitants of both cities and villages to reflect. In order for the world to prepare for the growing population of urbanized areas, it is necessary to look at and solve the problems that exist here now.

Implementation

Photoguard is a distributed system with 3-tier architecture including: Client presentation tier, logic servers tier and data server tier with connection to the database. First tier is written in HTML, CSS, JS and C# using Blazor framework and it is connected to second tier by RESTful API sending and receiving JSON files. Server to host images in the second tier is written in C# using Web API. App logic server is written in C#, but also communicates with third tier using gRPC framework which uses Protocol Buffers. Third tier data server is written in Java and communicates with PostgreSQL database with JDBC. Additionally, system uses Google Maps API.
